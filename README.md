# HyperFrames Rendering Server

A powerful video rendering server that accepts JSON timeline data and generates MP4 videos using HyperFrames.

## Overview

This project consists of:

1. **HyperFrames Composition Engine** — Renders video from HTML compositions
2. **JSON Timeline API** — Accepts structured timeline data
3. **Rendering Server** — Processes render jobs asynchronously

## Architecture

```
Client → POST /render (JSON) → Server
                ↓
          Parse Timeline
                ↓
          Generate HTML Composition
                ↓
          Run HyperFrames Render CLI
                ↓
          Output MP4 Video
```

## Quick Start

### Prerequisites

- Node.js >= 22 or Bun >= 1.0
- FFmpeg (auto-installed by HyperFrames)

### Installation

```bash
cd rendering-server
bun install  # or npm install
```

### Start the Server

```bash
# Using Bun (recommended)
bun run server.js

# Using Node.js
node server.js
```

Server starts on `http://localhost:3001`

## Usage

### Send a Render Job

```bash
curl -X POST http://localhost:3001/render \
  -H "Content-Type: application/json" \
  -d '{
    "id": "my-video",
    "duration": 5,
    "clips": [
      {
        "id": "title",
        "type": "text",
        "content": "Hello World",
        "start": 0,
        "duration": 5,
        "trackIndex": 0,
        "style": {
          "fontSize": "96px",
          "color": "#ffffff",
          "padding": "200px"
        }
      }
    ]
  }'
```

Response:
```json
{
  "jobId": "my-video-1234567890",
  "status": "rendering",
  "estimatedOutputPath": "renders/my-video-1234567890.mp4"
}
```

### Check Render Status

```bash
curl http://localhost:3001/status/my-video-1234567890
```

When complete:
```json
{
  "jobId": "my-video-1234567890",
  "status": "complete",
  "path": "renders/my-video-1234567890.mp4"
}
```

## Timeline JSON Schema

### Root Timeline Object

```json
{
  "id": "composition-id",
  "duration": 10,
  "width": 1920,
  "height": 1080,
  "background": "#000000",
  "audio": "path/to/audio.mp3",
  "clips": [...]
}
```

### Clip Objects

Clips can be text, images, or video:

**Text Clip:**
```json
{
  "id": "text-1",
  "type": "text",
  "content": "Hello",
  "start": 0,
  "duration": 5,
  "trackIndex": 0,
  "style": {
    "fontSize": "64px",
    "color": "#ffffff"
  }
}
```

**Image Clip:**
```json
{
  "id": "image-1",
  "type": "image",
  "content": "path/to/image.png",
  "start": 0,
  "duration": 5,
  "trackIndex": 0
}
```

**Video Clip:**
```json
{
  "id": "video-1",
  "type": "video",
  "content": "path/to/video.mp4",
  "start": 0,
  "duration": 5,
  "trackIndex": 0
}
```

## Advanced Features

### Using HyperFrames Skills

For complex animations beyond basic fades:

1. Edit `index.html` directly
2. Use the `/hyperframes` skill for:
   - GSAP animations (`/gsap`)
   - Three.js 3D (`/three`)
   - Lottie animations (`/lottie`)
   - CSS animations (`/css-animations`)
   - Audio-reactive visuals
   - Captions and TTS

3. Run the server to render with your custom composition

### Custom Styling

All CSS properties work in the `style` object (camelCase):

```json
{
  "style": {
    "fontSize": "48px",
    "fontWeight": "bold",
    "color": "#ffffff",
    "textAlign": "center",
    "backgroundColor": "rgba(0,0,0,0.5)",
    "padding": "40px",
    "borderRadius": "10px",
    "boxShadow": "0 4px 6px rgba(0,0,0,0.3)",
    "transform": "scale(1.1)",
    "opacity": "0.9"
  }
}
```

## Troubleshooting

### Server won't start

```bash
bun run server.js --port 3001
```

### HyperFrames not found

```bash
npm install hyperframes
```

### Render failures

Check HyperFrames environment:

```bash
npx hyperframes doctor
```

### Media files not found

Use absolute paths or place files in `assets/` directory.

## Performance

- **Quality Modes**: draft (fast), standard, high
- **Default**: draft for server renders (fast iteration)
- **Change quality** in `server.js` line 93: `--quality draft`
